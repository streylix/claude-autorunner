Issue: The .message-actions element has pointer-events: none by default, but when you hover over a message item, it becomes pointer-events: auto. This creates a "dead zone" where drag events are blocked.

Try this: Hover over a message to make the buttons visible, then try to drag from that area. The drag will fail because the actions element is capturing the mouse events.
